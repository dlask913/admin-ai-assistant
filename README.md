- 페이지, 메뉴, 상품 관련 질문 시 적합한 대답을 해주는 챗봇 구현
- 페이지 설명 / 메뉴 설명 → Vector, 실제 데이터 조회 → SQL (API)
- 잘 변하지 않는 정적 데이터는 Vector DB 로 관리하고 실시간으로 변하는 데이터는 API 를 통해 조회
- 예시 벡터 문서
```json
{
  "content": "재고 관리는 관리자 > 재고 관리 메뉴에서 상품별 수량을 확인할 수 있습니다.",
  "page_url": "/admin/inventory",
  "entity": "inventory"
}
```

### 워크플로우

```text
1. 사용자 질문
   ↓
2. Vector DB 검색
   → 어떤 '페이지/도메인' 질문인지 파악
   ↓
3. 해당 API 호출
   → 실시간 데이터 조회
   ↓
4. LLM이 결과 + 페이지 URL로 응답

```
